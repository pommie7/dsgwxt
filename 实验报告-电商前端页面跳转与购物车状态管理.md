# 实验报告：电商前端页面跳转与购物车状态管理

---

## 实验名称

**电商前端 — 404 页面跳转、下单成功跳转、Pinia 购物车全局状态管理**

---

## [实验任务与要求]

1. **页面跳转**：在电商前端工程中添加 404 页面（访问不存在的路径时展示）和下单成功页面（下单后跳转展示订单信息）
2. **全局状态管理**：配置 Pinia（使用 Zustand 实现 Pinia 模式），定义购物车全局状态，实现添加商品、删除商品、修改数量、计算总价功能
3. **购物车页面**：展示购物车列表，增加"清空购物车"按钮
4. **代码规范**：组件化开发，状态持久化，良好的 UI/UX

## [实验分析与设计]

### 一、页面跳转设计

```
用户访问路径
    │
    ├── /products          → ProductsPage (商品列表)
    ├── /products/:id      → ProductDetailPage (商品详情)
    ├── /cart              → CartPage (购物车)
    ├── /orders            → OrdersPage (订单列表)
    ├── /order-success     → OrderSuccessPage (下单成功)
    ├── /login /register   → 登录/注册页
    ├── /                  → 重定向到 /products
    └── /* (任意不存在路径) → NotFoundPage (404)
```

**404 页面设计**：
- 展示 404 错误码 + 当前访问路径
- 提供"返回首页"和"浏览商品"两个导航按钮
- 使用 `<Route path="*">` 匹配所有未定义路由

**下单成功页面设计**：
- 通过 URL query 参数接收 `orderNo` 和 `amount`
- 展示订单编号、支付金额
- 自动调用 `clearCart()` 清空购物车
- 8 秒倒计时自动跳转订单列表页
- 提供"查看订单"和"继续购物"按钮

### 二、购物车状态管理设计

**Pinia 模式**：Pinia 是 Vue 生态的状态管理库，本实验使用 Zustand 实现相同的 Store 模式。

```
┌──────────────────────────────────────────────────┐
│                  Pinia Store                      │
│  ┌─────────┐  ┌──────────┐  ┌────────────────┐  │
│  │  State  │  │  Getters  │  │    Actions     │  │
│  │ items[] │  │totalCount │  │ addItem()      │  │
│  │         │  │totalPrice │  │ removeItem()   │  │
│  │         │  │ isEmpty   │  │ updateQty()    │  │
│  │         │  │           │  │ clearCart()    │  │
│  └─────────┘  └──────────┘  └────────────────┘  │
│                     │                            │
│            localStorage 持久化                    │
└──────────────────────────────────────────────────┘
        │                    │
        ▼                    ▼
  ┌──────────┐       ┌──────────────┐
  │ CartPage │       │ Layout 角标   │
  │ 购物车页面│       │ 🛒 (N)       │
  └──────────┘       └──────────────┘
```

**数据结构**：
```js
{
  id: Date.now(),          // 购物车项唯一 ID
  productId: 1,            // 原始商品 ID
  name: "iPhone 15 Pro Max",
  price: 8499.15,          // 促销价（如有）
  quantity: 2,
  subtotal: 16998.30       // 小计
}
```

**状态持久化**：购物车数据自动同步到 `localStorage`，刷新页面不丢失。

## [实验总结]

### 实验收获

1. **SPA 路由设计**：掌握了 React Router v6 的路由配置、嵌套路由、动态参数、通配符匹配等核心用法
2. **全局状态管理**：理解了 Pinia/Zustand 的 Store 模式——State（数据）、Getters（计算属性）、Actions（修改方法）三层结构
3. **状态持久化**：学会了将全局状态同步到 localStorage 并在初始化时恢复
4. **组件间通信**：父→子通过 props 传值，跨组件通过全局 Store 共享
5. **页面跳转传参**：学会了通过 URL query string 在页面间传递参数

### 遇到的问题与解决

| 问题 | 解决方案 |
|------|----------|
| Pinia 是 Vue 生态库，不兼容 React | 使用 Zustand 实现相同的 Store 模式 |
| 编辑 JSX 时多了一个 `</div>` 导致构建失败 | 逐行核对标签闭合，删除多余标签 |
| GitHub 网络不稳定导致 push 失败 | 多次重试，选择网络畅通时机推送 |

### 注意事项

1. 全局状态库的选择要匹配框架（Vue→Pinia，React→Zustand/Redux）
2. localStorage 存储需用 try-catch 包裹，防止隐私模式下写入失败
3. JSX 中多个嵌套 div 时需注意闭合标签数量
4. 路由通配符 `*` 必须放在最后，否则会拦截所有路由
5. 购物车更新时需要同步更新 subtotal 小计字段
